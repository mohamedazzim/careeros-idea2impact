# Architecture Decision Record: ADR-004 — Why LangGraph for Multi-Agent Orchestrations

## Status
Approved

## Context
CareerOS AI's career agents must perform sequential, multi-step tasks (e.g., assessor profiles, grouping milestones, certification discovery, and tailored reviews) that depend on logical condition loops and human intervention. Linear chains are too rigid to handle these complex pipelines.

## Decision
LangGraph has been approved as our primary agent state machine executor based on the following capabilities:

1. **Flexible Cyclic Graphs (Cycles & Loops)**: Unlike DAG (Directed Acyclic Graph) frameworks, LangGraph allows agents to loop back to prior execution states (for example, self-correcting code or auto-tuning resumes based on validation failures).
2. **Robust State Persistence (Persistence & Checkpointing)**: Saves complete agent checkpoints at every node transition, allowing developers to restart interrupted runs and enabling administrators to inspect historic state transitions.
3. **First-Class Human-in-the-Loop Integration**: Native "interrupt/breakpoint" support halts agent executions before high-risk actions, securely transferring control to reviewers inside the Human Approval Center.
4. **Native Streaming support**: Supports streaming intermediate diagnostic tokens and node transitions directly to client terminals in real-time.

## Consequences
- **Positive**: Deterministic execution boundaries, resilient fault handling, and native human-in-the-loop controls.
- **Negative**: Steep learning curve and complex state schema tracking across graph transitions.
