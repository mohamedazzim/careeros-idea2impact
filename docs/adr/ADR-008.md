# Architecture Decision Record: ADR-008 — Why Human Approval Center (Human-in-the-Loop)

## Status
Approved

## Context
Deploying autonomous AI agents to interact with external systems (like LinkedIn messaging or Twilio SMS alerts) introduces risks of automated spam, hallucinated promises, formatting bugs, or inappropriate tone. This can damage candidate reputations or violate platform rules.

## Decision
CareerOS AI mandates a resilient, centralized Human-in-the-Loop (HITL) gateway called the **Human Approval Center**. No generated outbound communications can execute without reviewer approval:

1. **State-Locked Outbound Actions**: Outbound message generation actions write to an intermediary `approvals` table with a state of `PENDING`. Direct execution is blocked by database constraints.
2. **Review Interfaces**: Reviewers are provided with an intuitive UI highlighting diffs side-by-side to compare raw AI outputs against manual edits.
3. **Flexible Action Handles**: Reviewers can:
   - **Approve**: Set state to `APPROVED`, releasing the message to outbound MCP queues.
   - **Edit & Approve**: Set state to `EDITED`, saving manual corrections before execution.
   - **Reject**: Set state to `REJECTED`, logging reasons and blocking the run.
4. **Permanent Audit Trail**: Stores all action records (Reviewer ID, action type, changes, timestamps) to ensure a complete audit history.

## Consequences
- **Positive**: Complete security coverage against LLM hallucinations, high outbound quality control, and zero risk of automated spam.
- **Negative**: Adds human friction to the process, requiring reviewer action before messages can be sent.
