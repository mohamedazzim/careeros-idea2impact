# Architecture Decision Record: ADR-001 — Why FastAPI for Backend Services

## Status
Approved

## Context
For enterprise artificial intelligence services, CareerOS AI requires high-performance, asynchronous orchestration loops capable of processing concurrent multi-agent graph flows, holding high-fidelity vector matches, and streaming live token bytes to candidate clients. We evaluated Node Express, Django Channels, and FastAPI.

## Decision
FastAPI was chosen as the enterprise architectural standard for CareerOS AI's pythonic service layers due to several criteria:

1. **Native Asynchronous Event Loops**: FastAPI is built directly on top of Starlette and Uvicorn, registering performance parameters comparable to Go and NodeJS while remaining fully expressive in pythonic AI libraries (LangGraph, PyTorch, NumPy).
2. **Standardized OpenAPI Generation**: Automatically reads Pydantic type models and serves rigorous interactive Swagger or Redoc endpoints with zero manual schema annotations.
3. **Structured Pydantic Validations**: Eliminates injection vulnerabilities at ingress gates by validating string, integer, float, and complex nested coordinate models cleanly with optimized C-compiled type engines.
4. **Native Dependency Injection**: Fully supports clean, nested mock declarations during continuous integration tests.

## Consequences
- **Positive**: Near-zero parsing overhead. Fully automated, compliant OpenAPI document endpoints are immediately available for external developers.
- **Negative**: Requires python asynchronous programming (using `async` / `await` paradigms), which demands careful handling of non-async blocked libraries using event loop background executors.
