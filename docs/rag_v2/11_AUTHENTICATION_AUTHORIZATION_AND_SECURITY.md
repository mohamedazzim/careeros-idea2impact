# Authentication, Authorization, And Security

CareerOS authentication is implemented with password hashing, JWT token pairs, HttpOnly cookies, bearer token support, lockout after failed login attempts, audit logging, and ownership checks in route handlers.

## Auth Flow

`POST /api/v1/auth/register` validates email and password strength, creates a user with bcrypt or PBKDF2 fallback hashing, and returns/sets token cookies. `POST /api/v1/auth/login` verifies credentials, enforces account lockout after five failed attempts for fifteen minutes, resets failed attempt state on success, and returns/sets token cookies. `GET /api/v1/auth/me`, `PATCH /api/v1/auth/me`, password change, logout, reset password, and account deletion are implemented.

## Security Controls

- Passwords require at least 12 characters plus uppercase, lowercase, digit, and special character.
- Tokens are generated by `auth_service` and can be revoked.
- Cookies are `HttpOnly`, `SameSite=Lax`, and secure outside debug.
- Production startup validation rejects weak secrets and localhost infrastructure values.
- AI prompt-injection and secret-seeking checks are enforced for docs-RAG questions.
- Uploads reject unextractable resume text for PDF/DOCX resume files.
